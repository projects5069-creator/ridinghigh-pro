## חוקי עבודה נוספים שתמיד תקפים

(תזכורת — אלה כללים כלליים שתמיד תקפים, לא רק בפתיחת סשן)

1. **§10 Single Source of Truth** — כל מטריקה מחושבת במקום אחד בלבד
2. **§12 Versioned filenames** — לעולם לא לדרוס קבצים קיימים, להשתמש ב-_v1, _v2 וכו'
3. **Commit protocol** — backup → str_replace patch → py_compile → diff review → אישור מפורש → commit + push
4. **Anti-Drift Maintenance Contract** — לפני commit שנוגע בנוסחאות/משקלים/workflows: לעדכן PK_v2 ולהגדיל גרסה
5. **Iron Rule — Always verify, never assume** — כל קביעה חייבת מקור מאומת

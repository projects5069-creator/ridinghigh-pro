# Health Audit System — RidingHigh Pro

Automated health monitoring that runs **3 times a day** and writes results to a dedicated Google Sheet.

## What it checks (19 checks)

### 📁 Code integrity (3)
- **C1** — Duplicate function definitions across files (single source of truth)
- **C2** — Hardcoded thresholds that should be in `config.py` (`0.07`, `0.10`, `>=60`, `>=70`)
- **C3** — `dashboard.py` properly imports from `formulas.py` and `config.py`

### 📊 Data freshness (3)
- **D1** — `timeline_live` updated within 24h on trading days
- **D2** — `post_analysis` has rows for last 3 trading days
- **D3** — GitHub Actions success rate ≥95% in last 24h

### 🔢 Data quality (4)
- **Q1** — All Scores in `[0, 100]` range
- **Q2** — All sheets have required columns (Date, Ticker, Score, etc.)
- **Q3** — No duplicate `(Ticker, ScanDate)` in `post_analysis`
- **Q4** — No outliers (REL_VOL > 100, ScanPrice ≤ 0)

### ⚙️ Config consistency (3)
- **X1** — `sheets_config.json` has entry for current Peru month
- **X2** — `SCORE_WEIGHTS_V2` sums to 100 (or 1.0)
- **X3** — All critical files exist (auto_scanner, dashboard, etc.)

### 🔐 Repo health (2)
- **R1** — Number of uncommitted files (warning at >5, critical at >15)
- **R2** — No tracked files matching `.gitignore` patterns (`_BEFORE_`, `.pyc`, etc.)

### 🔄 Sync (1)
- **19** — Local repo sync with `origin/main`: detects unpushed commits (escalates by age: INFO <6h, WARNING 6-24h, CRITICAL >24h), behind/diverged states, and stale issues in `OPEN_ISSUES.md` (marked fixed but still in STILL OPEN section). Skipped in CI.

## Schedule

| Peru time | UTC | Purpose |
|-----------|-----|---------|
| 06:00 | 11:00 | Pre-market check |
| 12:00 | 17:00 | Mid-day check |
| 22:00 | 03:00 (next day) | End-of-day check |

Runs **every day** including weekends — scanning is paused but checks still detect drift.

## Output

Results written to **`RidingHigh-Health-Audit`** Google Sheet with 3 tabs:

- **History** — Append every run. Long-term trend tracking.
- **Latest** — Overwritten each run. Current state at a glance.
- **Failed** — Only currently-failing checks. Quick triage view.

## Severity levels

| Status | Meaning | Exit code |
|--------|---------|-----------|
| 🔴 CRITICAL | Affects data accuracy or scanner operation | 1 |
| 🟠 WARNING | System works but suboptimal | 2 (only warnings) |
| 🔵 INFO | Notable but not an issue | — |
| ✅ PASSED | All good | 0 |

## Manual run (local)

```bash
cd ~/RidingHighPro

# Run with full output to console + write to Sheet
python3 health_audit.py --local

# Just print, don't write to Sheet
python3 health_audit.py --local --no-sheet
```

## Manual trigger (GitHub Actions)

```
1. Go to https://github.com/projects5069-creator/ridinghigh-pro/actions
2. Click "Health Audit" workflow
3. Click "Run workflow" → "Run workflow"
```

## Files

| File | Role |
|------|------|
| `health_audit.py` | Main audit script (19 checks) |
| `check_sync.py` | Standalone quick sync check (runs only check_19) |
| `.github/workflows/health_audit.yml` | GitHub Actions cron schedule |
| `setup_health_audit_sheet.py` | One-time Sheet creation (locally) |
| `.health_audit_sheet_id` | Generated by setup — gitignored |

## Required GitHub Secrets

The workflow expects these secrets:
- `GOOGLE_CREDENTIALS_JSON` — Service account JSON (already configured)
- `GITHUB_TOKEN` — Auto-provided by GitHub Actions

## Modifying checks

To add/remove a check:
1. Edit `health_audit.py`
2. Add/remove a `check_NN_*()` function
3. Add the function call in `main()`
4. Push to GitHub — next run picks up the change

## Troubleshooting

- **No data in Sheet** — Check that `.health_audit_sheet_id` exists. Re-run `setup_health_audit_sheet.py` if needed.
- **Auth errors in CI** — Verify `GOOGLE_CREDENTIALS_JSON` secret is set correctly in repo settings.
- **D3 shows "API query failed"** — GitHub API rate limits hit. Pass `GITHUB_TOKEN` to fix.
